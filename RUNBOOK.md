# Как прогонять — коротко

Ты делаешь только это: заходишь в папку, запускаешь в ней своего агента, вставляешь
промпт, ждёшь. Оценку, цифры и ревью делаю я — потом, из этих же папок.

## Кто в какой папке
В каждой папке запускай СВОЕГО агента (эта папка = его рабочая директория):

| папка | агент |
|---|---|
| `…/codex56` | Codex · gpt-5.6-sol · medium |
| `…/ccx`     | ccx (Claude Code + gpt-5.6-sol) · medium |
| `…/ccopus`  | нативный Claude Code · opus-4.8 · xhigh |
| `…/codex55` | Codex · gpt-5.5 · high |

## Порядок: 2 прогона × 6 задач × 4 окна
Прогон 1 — все папки лежат в `runs/<задача>/r1/…`, прогон 2 — в `…/r2/…`.
Папки уже созданы, ничего создавать не надо.

Для каждой задачи (пример — t1, прогон 1):
1. Открой 4 окна, в каждом зайди в свою папку:
   - `runs/t1-py-bugfix/r1/codex56`
   - `runs/t1-py-bugfix/r1/ccx`
   - `runs/t1-py-bugfix/r1/ccopus`
   - `runs/t1-py-bugfix/r1/codex55`
2. Запусти в каждом окне своего агента.
3. Вставь во все 4 ОДИН И ТОТ ЖЕ промпт — содержимое `tasks/t1-py-bugfix/PROMPT.md`
   (быстро глянуть: `cat tasks/t1-py-bugfix/PROMPT.md`).
4. Enter. Жди, пока агент сам закончит.
5. Все 4 готовы → следующая задача.

Задачи по порядку: t1-py-bugfix · t2-ts-refactor · t3-ts-feature · t4-diagnosis ·
t5-diff-bugfix · t6-ts-feature. Потом повтори всё с папками `…/r2/…`.

## Важно
- Запускай агента ИМЕННО внутри его папки — иначе я не свяжу результат с колонкой.
- Ничего не чини руками и не подсказывай агенту — пусть работает сам, это и есть замер.
- Никакие скрипты запускать не надо. Закончил прогоны — отдал мне репо, дальше я.
