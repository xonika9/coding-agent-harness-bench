# Спецификация: движок запросов к записям (`run_query`)

Реализуй в `src/query.py`:

```python
def run_query(rows: list[dict], spec: dict) -> list[dict]:
    ...
```

`rows` — список записей (каждая — `dict` с произвольными полями). `spec` — описание
запроса. Функция применяет к `rows` конвейер стадий и возвращает новый список записей.

Это НЕ SQL и НЕ pandas — семантика ниже единственный источник истины. Следуй ей буквально.

## Порядок стадий (фиксированный)

```
filter -> sort -> distinct -> paginate -> project
```

Каждая стадия опциональна: если соответствующего ключа в `spec` нет — стадия пропускается.
Исходный `rows` не мутируется. Возвращаются новые dict-и (записи не разделяют ссылок с входом).

## Ошибки

Некорректный `spec` или несопоставимые данные → `raise ValueError`. Конкретные случаи
перечислены у каждой стадии. Никаких «тихих» подмен: не глотай ошибку сравнения типов,
не приводи типы, не угадывай намерение.

## 1. filter — `spec["where"]`

Список условий; запись проходит, если выполнены **ВСЕ** (AND). Условие:
`{"field": str, "op": str, "value": ...}`.

- Если `field` отсутствует в записи — условие **ложно** (запись отсеивается), для любого
  оператора, без ошибки.
- Операторы:
  - `"=="`, `"!="` — равенство/неравенство. Разные типы не равны (`5 == "5"` → ложно).
  - `"<"`, `"<="`, `">"`, `">="` — порядковое сравнение. Требуется, чтобы значение поля и
    `value` были **одного порядкового вида**: оба числа (`int`/`float`, но **`bool` числом
    не считается**) ИЛИ обе строки. Иначе (число со строкой, `bool`, или несравнимый
    тип) → `raise ValueError`.
  - `"in"` — `value` обязано быть списком/кортежем; условие истинно, если значение поля
    содержится в `value`. Если `value` не список/кортеж → `raise ValueError`.
  - `"contains"` — значение поля должно быть строкой или списком; истинно, если `value`
    содержится в нём. Если значение поля не строка и не список → условие ложно (без ошибки).
  - Любой другой `op` → `raise ValueError`.

## 2. sort — `spec["order_by"]`

Список ключей: `{"field": str, "dir": "asc" | "desc"}`. `dir` по умолчанию `"asc"`;
любое другое значение `dir` → `raise ValueError`. Сортировка **устойчивая**,
многоключевая (первый ключ — старший).

- Если у записи нет поля ключа — считается, что значение `None`.
- `None` считается **меньше** любого не-`None`. При `"asc"` `None` идут первыми, при
  `"desc"` — последними.
- Среди не-`None` значений одного ключа все должны быть одного порядкового вида (все числа
  или все строки). Если по одному ключу встречаются и число, и строка → `raise ValueError`.

## 3. distinct — `spec["distinct"]`

Если значение истинно (`True`) — убрать дубликаты записей, сравнивая записи по полному
содержимому (равные dict-и), сохраняя порядок первого вхождения.

## 4. paginate — `spec["offset"]`, `spec["limit"]`

Сначала `offset` (по умолчанию `0`), затем `limit` (по умолчанию — без ограничения).
`offset` и `limit`, если заданы, должны быть неотрицательными `int` (тип `bool` не считается
`int`); иначе → `raise ValueError`. `offset` за пределами длины → пустой список.

## 5. project — `spec["select"]`

- Список имён полей `["a", "b"]` → каждая запись превращается в dict только с этими
  ключами; если поля нет в записи, значение ключа — `None`.
- Dict `{"out": "in", ...}` → переименование: ключ результата `out` берёт значение поля `in`
  (если поля нет — `None`).
- Если `select` отсутствует — записи возвращаются как есть (новые dict-копии).
