# Спецификация: стековая виртуальная машина (`run`)

Реализуй в `src/vm.ts`:

```ts
export interface VMResult { output: number[]; stack: number[]; }
export function run(program: string): VMResult;
```

`program` — исходный текст программы для маленькой стековой VM (одна инструкция на строку).
`run` загружает программу, исполняет её и возвращает `{ output, stack }`, где `output` —
последовательность напечатанных значений (в порядке печати), а `stack` — содержимое
операндного стека на момент останова, снизу вверх (последний добавленный — последний в
массиве).

Это НЕ реальный ассемблер и НЕ какой-либо существующий байткод — единственный источник
истины эта спека. Следуй ей буквально.

## Значения

Все значения на стеке и в переменных — **целые числа** (могут быть отрицательными).
Вся арифметика целочисленная. Никаких чисел с плавающей точкой в результатах не возникает.

## Формат программы

Программа — это текст, разбитый на строки (`\n`). Обработка строки:

1. Всё, начиная с символа `#` и до конца строки, — комментарий, отбрасывается.
2. Ведущие и хвостовые пробелы/табы отбрасываются.
3. Пустая после этого строка игнорируется.
4. Строка вида `имя:` (имя, затем двоеточие, и больше ничего значащего) — **определение
   метки** (label). Имя метки: `[A-Za-z_][A-Za-z0-9_]*`. Двойное определение одной метки →
   ошибка.
5. Иначе строка — **инструкция**: первый токен (разделитель — пробелы/табы) это опкод,
   остальные токены — операнды.

Имена меток и переменных **регистрозависимы** (`x` и `X` — разные). Опкоды
**регистронезависимы** (`ADD`, `add`, `Add` — одно и то же).

## Загрузка и валидация (до исполнения)

Программа сначала полностью разбирается и валидируется, и только потом исполняется. На этом
этапе бросается ошибка (`throw`), если:

- неизвестный опкод;
- у инструкции неверное число/вид операндов (см. таблицу);
- операнд-метка ссылается на несуществующую метку (проверяется для всех `JMP/JZ/JNZ`, даже
  если инструкция при исполнении недостижима);
- двойное определение метки;
- у `PUSH` операнд не является целым числом (допускается необязательный ведущий `-`).

Пустая программа (нет ни одной инструкции) — валидна: `run` возвращает `{ output: [], stack: [] }`.

## Инструкции

Бинарные операции берут операнды со стека так: если вершина стека `[..., a, b]` (b — верхний),
то они снимают `b`, затем `a`, и кладут результат `a ∘ b`.

| Опкод | Операнд | Действие |
|---|---|---|
| `PUSH n` | целое `n` | положить `n` |
| `POP` | — | снять и отбросить вершину |
| `DUP` | — | продублировать вершину |
| `SWAP` | — | поменять местами два верхних |
| `OVER` | — | положить копию второго сверху (`[a,b]` → `[a,b,a]`) |
| `ADD` | — | `a + b` |
| `SUB` | — | `a - b` |
| `MUL` | — | `a * b` |
| `DIV` | — | целочисленное деление, усечение **к нулю**; `b == 0` → ошибка |
| `MOD` | — | остаток `a - trunc(a/b)*b` (знак как у `a`); `b == 0` → ошибка |
| `NEG` | — | заменить вершину `a` на `-a` |
| `EQ` | — | `1` если `a == b`, иначе `0` |
| `NE` | — | `1` если `a != b`, иначе `0` |
| `LT` | — | `1` если `a < b`, иначе `0` |
| `LE` | — | `1` если `a <= b`, иначе `0` |
| `GT` | — | `1` если `a > b`, иначе `0` |
| `GE` | — | `1` если `a >= b`, иначе `0` |
| `NOT` | — | снять `a`; `1` если `a == 0`, иначе `0` |
| `AND` | — | снять `b`,`a`; `1` если оба ненулевые, иначе `0` |
| `OR` | — | снять `b`,`a`; `1` если хотя бы один ненулевой, иначе `0` |
| `LOAD name` | имя | положить значение переменной `name` |
| `STORE name` | имя | снять вершину и записать в переменную `name` |
| `PRINT` | — | снять вершину и дописать её в `output` |
| `JMP label` | метка | безусловный переход |
| `JZ label` | метка | снять вершину; если она `== 0`, перейти |
| `JNZ label` | метка | снять вершину; если она `!= 0`, перейти |
| `HALT` | — | немедленно завершить исполнение |

Имя переменной: `[A-Za-z_][A-Za-z0-9_]*`.

## Исполнение

Исполнение начинается с первой инструкции (в порядке текста; метки не занимают «шага») и идёт
последовательно, пока не будет достигнут конец программы, либо не выполнится `HALT` — тогда
`run` возвращает текущие `output` и `stack`.

Ошибки исполнения (`throw`):

- **stack underflow**: инструкции не хватает операндов на стеке;
- **деление/остаток на ноль** (`DIV`/`MOD` при `b == 0`);
- **неинициализированная переменная**: `LOAD name`, когда в `name` ещё ничего не записано;
- **лимит шагов**: если исполнено более `100000` инструкций — бросить ошибку (защита от
  зацикливания). Метки шагами не считаются; считается каждая исполненная инструкция.

Все ошибки — это `throw` (тип не важен; тесты проверяют только факт броска). «Тихих» подмен
быть не должно: не глотай underflow, не превращай деление на ноль в `0`, не читай
неинициализированную переменную как `0`.

## Проверка

- Тесты: `node --test 'test/**/*.test.ts'` — должны стать зелёными.
- Типы: `tsc --noEmit -p tsconfig.json` — чисто.
- Решение проверяется в том числе на **скрытых** крайних случаях из этой спеки, которых нет
  в видимых тестах. Следуй спецификации буквально, а не только видимым примерам.
- Node исполняет `.ts` в режиме strip-only: **нельзя** использовать TypeScript-конструкции,
  требующие кодогенерации — `enum`, parameter properties (`constructor(private x)`),
  `namespace`. Используй обычные типы и классы с явными полями.
