# Спецификация: мини-DSL фильтра

Реализуй в `src/filter.ts` две связанные функции.

## Типы
- `type Row = Record<string, string | number>`
- `type Op = ">=" | "<=" | "!=" | "=" | ">" | "<"`
- `interface Condition { field: string; op: Op; value: string | number }`

## `parseQuery(query: string): Condition[]`
- Условия разделяются строкой ` AND ` (в верхнем регистре, с пробелами вокруг).
- Условие: `field OP value`.
  - `field` — начинается с буквы или `_`, дальше буквы/цифры/`_`.
  - `OP` — один из шести операторов выше.
  - `value` — остаток после оператора, обрезанный по краям. Если значение выглядит
    как число (`-?\d+(\.\d+)?`) — это число, иначе строка.
- Пробелы вокруг field/op/value и по краям всего запроса игнорируются.
- Невалидный запрос (пустой, нет оператора, пустое значение, пустой/битый field,
  неизвестный оператор) — бросать `Error`.

## `applyFilter(rows: Row[], query: string): Row[]`
- Возвращает строки, где выполнены ВСЕ условия (AND).
- `=` и `!=` сравнивают по строковому представлению значений.
- `>`, `>=`, `<`, `<=` — числовое сравнение; если любое значение не число — условие ложно.
- Если поля нет в строке — условие ложно (строка не проходит).
