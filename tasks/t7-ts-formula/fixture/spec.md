# Спецификация: мини-движок формул таблицы (`evalFormula`)

Реализуй в `src/formula.ts`:

```ts
export function evalFormula(formula: string, cells: Record<string, number | string>): number | string
```

Вычисляет формулу таблицы (как в электронной таблице) по карте значений ячеек `cells`
и возвращает результат: число, строку-значение, **или строку-ошибку** (`#DIV/0!`,
`#VALUE!`, `#NAME?`).

Это НЕ Excel. Семантика ниже — единственный источник истины; следуй ей буквально,
а не привычкам Excel/Google Sheets.

## Две категории неправильного

Различай строго:

1. **Структурная ошибка входа → `throw new Error(...)`.** Это формула, которую нельзя
   было написать (синтаксис/арность). Список — в разделе «Структурные ошибки».
2. **Ошибка-значение → верни строку `#DIV/0!` / `#VALUE!` / `#NAME?`** (НЕ бросай).
   Это корректно разобранная формула, но с ошибкой вычисления. Ошибки-значения
   **распространяются** (см. «Распространение»).

Перепутать эти две категории — ошибка. Например, деление на ноль — это `#DIV/0!`
(значение), а незакрытая скобка — это `throw`.

## Токены

- **Число**: `\d+(\.\d+)?` (неотрицательное; знак минус — это оператор, а не часть числа).
- **Строка-литерал**: `"..."` в двойных кавычках, может содержать пробелы; экранирования
  нет (внутри кавычек `"` не встречается).
- **Ссылка на ячейку**: `[A-Z]+[0-9]+` (заглавные буквы, затем цифры), напр. `A1`, `AB12`.
  Значение берётся из `cells[ref]`. **Если ключа нет в `cells` — ячейка пустая, значение `0`.**
- **Диапазон**: `REF:REF`, где обе ссылки в одном столбце (одинаковые буквы: `A1:A5`)
  ИЛИ в одной строке (одинаковые цифры: `A1:E1`). Разворачивается в список ячеек по
  порядку (по возрастанию). Диапазон допустим **только как непосредственный аргумент
  функции**. Диапазон с разными буквами И разными цифрами (`A1:B2`) — структурная ошибка.
- **Операторы**: `+ - * / ^`, скобки `( )`, запятая `,` (разделитель аргументов функции).
- Пробелы между токенами игнорируются.

## Операторы и приоритет (от высшего к низшему)

1. `^` — возведение в степень, **право-ассоциативный** (`2^3^2` = `2^(3^2)` = 512).
2. унарный `-` (напр. `-A1`, `-3`).
3. `*` `/`.
4. `+` `-` (бинарный).
5. сравнения `=` `<>` `<` `<=` `>` `>=` — **лево-ассоциативные**, ниже арифметики.

Арифметические операторы (`^ * / + -`, унарный `-`) работают **только с числами**.
Если операнд — строка (строка-литерал или ячейка со строковым значением) — результат
`#VALUE!`. Деление `x/0` (и `/` с нулевым делителем) — `#DIV/0!`.

Сравнения возвращают число **`1`** (истина) или **`0`** (ложь):
- `<` `<=` `>` `>=` — только числа; если любой операнд не число (строка) → `#VALUE!`.
- `=` `<>` — сравнивают по значению: два числа — численно; две строки — по равенству
  строк; **число со строкой (разные типы) — не равны** (`=` → `0`, `<>` → `1`).

## Функции

Имена функций — **строго заглавными**. Неизвестное имя (в т.ч. в нижнем регистре,
напр. `sum`) → ошибка-значение `#NAME?`.

- `SUM(a, b, ...)` — сумма. Аргументы — скаляры и/или диапазоны. ≥ 1 аргумент.
- `AVG(a, b, ...)` — среднее = сумма / количество значений. ≥ 1 аргумент. Если значений
  ноль — `#DIV/0!` (пустых диапазонов не бывает, но `AVG()` без аргументов — структурная
  ошибка арности, а не `#DIV/0!`).
- `MIN(a, b, ...)`, `MAX(a, b, ...)` — минимум/максимум. ≥ 1 аргумент.
- `ABS(x)` — модуль. Ровно 1 аргумент.
- `ROUND(x, n)` — округление `x` до `n` знаков после запятой, **half away from zero**
  (`ROUND(2.5,0)=3`, `ROUND(-2.5,0)=-3`, `ROUND(1.2345,2)=1.23`). Ровно 2 аргумента;
  `n` должно вычисляться в **целое число** (иначе `#VALUE!`).
- `IF(cond, a, b)` — если `cond` ≠ 0, вернуть `a`, иначе `b`. Ровно 3 аргумента. `cond`
  должен быть числом; если `cond` — строка, результат `#VALUE!`. Ветви `a`/`b` — любые
  (число/строка/ошибка); вычисляется и возвращается только выбранная ветвь-значение
  (но обе ветви должны быть синтаксически корректны — они разбираются всегда).
- `AND(a, b, ...)`, `OR(a, b, ...)` — логические. ≥ 1 аргумент, **только скаляры**
  (диапазон → структурная ошибка). Каждый аргумент — число (0 = ложь, ≠0 = истина);
  строка → `#VALUE!`. `AND` → `1`, если все ≠ 0, иначе `0`. `OR` → `1`, если хотя бы
  один ≠ 0, иначе `0`.
- `NOT(x)` — ровно 1 аргумент, число; `1`, если `x = 0`, иначе `0`. Строка → `#VALUE!`.
- `COUNT(a, b, ...)` — ≥ 1 аргумент, скаляры и/или диапазоны. Возвращает **количество
  значений, являющихся числами** (пустая ячейка = `0` = число, считается). Строковые
  значения НЕ считаются и НЕ дают `#VALUE!` — они просто пропускаются. Ошибки-значения
  среди аргументов распространяются.

Числовые функции (`SUM/AVG/MIN/MAX/ABS/ROUND`) требуют, чтобы все используемые значения
были числами. Строковое значение среди них → `#VALUE!`. Пустая ячейка = `0` (число).

**Диапазоны допустимы только как аргумент `SUM/AVG/MIN/MAX/COUNT`.** Диапазон, переданный
в `ABS/ROUND/IF/AND/OR/NOT`, — структурная ошибка.

## Распространение ошибок-значений

Если любой операнд оператора или любое значение, используемое функцией, уже является
ошибкой-значением, результат — **эта же** ошибка-значение (при нескольких — самая левая
по порядку вычисления). Пример: `=A1 + 1` при `A1 = "#DIV/0!"`? Нет — строковое значение
ячейки `"#DIV/0!"` это ОБЫЧНАЯ строка, а не ошибка (→ `#VALUE!` в арифметике). Ошибки-значения
возникают только при вычислении (деление на ноль, тип, неизвестная функция), а затем
распространяются вверх по дереву. Строки в `cells`, буквально равные `"#DIV/0!"`, ошибками
не считаются.

## Результат верхнего уровня

- Если формула вычислилась в число — вернуть `number`.
- Если в строку-значение (напр. `="hello"` или `=IF(1,"a","b")`) — вернуть эту `string`.
- Если в ошибку-значение — вернуть строку ошибки (`#DIV/0!` / `#VALUE!` / `#NAME?`).

## Структурные ошибки (→ `throw new Error`)

- Пустая формула или только пробелы.
- Формула не начинается с `=`.
- Небалансированные скобки; лишние токены в конце; пропущенный операнд/оператор.
- Непонятный символ/токен (напр. `=A1 & B1`, `=@`, `=1..2`).
- Диапазон вне позиции аргумента функции (`=A1:A3`, `=A1:A3 + 1`).
- Диапазон с разными буквами и цифрами одновременно (`=SUM(A1:B2)`).
- Неверная арность известной функции (`=IF(1,2)`, `=ABS()`, `=ROUND(1)`, `=SUM()`).
- Вызов «функции» без имени или число как функция (`=1(2)`).

Неизвестное **имя функции** — это НЕ структурная ошибка, а `#NAME?` (значение).
