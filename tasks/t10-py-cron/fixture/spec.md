# Спецификация: планировщик cron (`next_runs`)

Реализуй в `src/cron.py`:

```python
def next_runs(expr: str, after: str, count: int) -> list[str]:
    ...
```

`expr` — cron-выражение из **пяти** полей. `after` — момент времени в ISO-формате. `count` —
сколько ближайших срабатываний вернуть. Функция возвращает список из `count` строк — ближайшие
моменты (строго позже `after`), в которые расписание срабатывает, в хронологическом порядке.

Время наивное (без часовых поясов и перехода на летнее/зимнее). Секунды в расписании не
участвуют — срабатывания всегда в начале минуты.

Это НЕ обёртка над системным cron и не обязана совпадать с какой-то конкретной реализацией во
всех деталях — единственный источник истины эта спека. Следуй ей буквально.

## Формат выражения

Пять полей, разделённых пробелами (любое число пробелов между полями):

```
минуты  часы  день-месяца  месяц  день-недели
```

Диапазоны значений:

| Поле | Диапазон | Имена |
|---|---|---|
| минуты | 0–59 | нет |
| часы | 0–23 | нет |
| день месяца (dom) | 1–31 | нет |
| месяц | 1–12 | `JAN`…`DEC` (регистр не важен) |
| день недели (dow) | 0–7, где 0 и 7 = воскресенье | `SUN`…`SAT` (0=`SUN`…6=`SAT`) |

Не пять полей → `raise ValueError`.

### Синтаксис одного поля

Поле — это список термов через запятую (объединение множеств). Пустое поле или пустой элемент
списка (`1,,2`) → `ValueError`. Терм — одно из:

- `*` — все значения диапазона поля.
- `n` — одиночное значение.
- `a-b` — включительный диапазон, `a <= b` (иначе `ValueError`).
- `*/k` — шаг `k` по всему диапазону: `low, low+k, low+2k, …`.
- `a-b/k` — шаг `k` внутри `a-b`.
- `a/k` — шаг `k` от `a` до максимума диапазона поля (например, минуты `5/15` → `5,20,35,50`).

`k` — целое ≥ 1 (без знака). `k <= 0`, `*/0`, дробный или нечисловой шаг, лишние `/`
(`1/2/3`) → `ValueError`.

Имена (`JAN`, `MON`, …) допустимы **только** в полях месяца и дня недели, в том числе внутри
диапазона (`MON-FRI`, `JAN-MAR/2`). Имя в числовом поле, неизвестное имя, значение вне
диапазона, мусорный терм (`-5`, `1-`, `1-2-3`, `a..b`) → `ValueError`.

Для дня недели `7` эквивалентно `0` (воскресенье). Диапазоны вроде `0-7` допустимы.

## Правило дня месяца / дня недели

Поле считается «звёздным» тогда и только тогда, когда его текст — ровно `*` (не `*/1`, не
`0-6`, не `1-31`). Календарный день подходит под расписание, если:

- оба поля (dom и dow) **не** звёздные → день подходит, когда совпадает dom **ИЛИ** dow;
- звёздное только dow → смотрим только dom;
- звёздное только dom → смотрим только dow;
- оба звёздные → подходит любой день.

День недели вычисляется по календарю: воскресенье = 0, …, суббота = 6.

## Срабатывание

Момент времени подходит, когда его минута ∈ множеству минут, час ∈ множеству часов, месяц ∈
множеству месяцев и календарный день подходит под правило dom/dow выше.

Несуществующие даты не рождаются сами: например `31` в 30-дневном месяце или `30 2` (30 февраля)
просто никогда не совпадут.

## Формат `after` и результата

- `after` парсится как ISO-8601: `YYYY-MM-DDTHH:MM` или `YYYY-MM-DDTHH:MM:SS`. Невалидный
  `after` → `ValueError`.
- Возвращаются моменты **строго позже** `after`. Практически: первый кандидат — начало
  следующей минуты после `after` (минута самого `after` не рассматривается, даже если секунды
  равны нулю).
- Каждый результат — строка `YYYY-MM-DDTHH:MM` с нулями в разрядах (например `2026-07-15T00:05`).

## Ограничения

- `count` — целое ≥ 1; иначе `ValueError`.
- Поиск ограничен горизонтом в **4 года** от `after`. Если за это время не набирается `count`
  срабатываний (расписание невыполнимо, например `0 0 30 2 *`, или слишком редкое) →
  `ValueError`.
